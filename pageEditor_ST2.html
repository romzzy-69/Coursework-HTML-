Splash Screen

Technical Description

The Splash Screen is the first page users see when visiting the BlueFuture website. It introduces the site's identity, mission, and team before automatically redirecting to the Home Page.

HTML Structure: The page uses a single <div class="overlay"> that covers the full screen with a dark transparent layer over the background image. Inside is a <div class="content"> which holds the logo image, an <h1> for the site title, an <h2> for the mission statement, a team card using CSS grid, and the countdown row with the skip button.

CSS Techniques: The body uses background-image with background-size: cover and height: 100vh to make the ocean image fill the entire screen. The overlay uses display: flex with justify-content: center and align-items: center to centre all content both vertically and horizontally. The team members section uses grid-template-columns: 1fr 1fr for a two column layout. The CSS loader spinner uses @keyframes spin with transform: rotate to create a spinning animation without any images or JavaScript.

JavaScript: A setInterval function runs every 1000 milliseconds and decreases a seconds variable by 1 on each tick. It updates the countdown text using textContent. When the counter hits 0 it calls clearInterval to stop the timer and uses window.location.href to redirect to the Home Page. A <meta http-equiv="refresh"> tag is also included as a fallback redirect in case JavaScript is disabled in the browser.

Accessibility

  • The logo image includes descriptive alt text so screen readers can identify it
  • The lang="en" attribute is set on the <html> tag to help screen readers identify the language
  • The Skip Intro button is a standard <a> tag so it is fully keyboard accessible
  • White text on a dark overlay ensures sufficient colour contrast for readability
  • Text shadow on the heading improves legibility against the background image

Action Impact Simulator

Technical Description

The Action Impact Simulator is an interactive page that lets users select ocean-friendly actions and receive live feedback on their total environmental impact score.

HTML Structure: The page uses the shared team template with a sticky header, a frosted overlay div containing the page content, and the shared footer. The main content is split into three parts a hero section with the title, a two column card grid with six action cards, and a summary box at the bottom. Each card has an image wrapper with an absolutely positioned points badge, a radio circle div, a heading and a short description. A data-points attribute on each card stores the point value so JavaScript can read it without hardcoding values in the script.

CSS Techniques: The background image is applied to the body with a transition property so it changes smoothly when the impact level updates. A div.ais-overlay uses background-color: rgba(255,255,255,0.55) to create a frosted glass effect over the background. The card grid uses display: grid with grid-template-columns: repeat(2, 1fr) for the two column layout. The points badge uses position: absolute inside a position: relative wrapper to pin it to the bottom left corner of the image. The card body uses display: flex to place the radio circle and text side by side.

JavaScript: document.querySelectorAll('.ais-card') grabs all six cards into a list. A click event listener added with addEventListener runs on each card. Clicking calls classList.toggle('selected') to add or remove the selected state. The updateScore function loops through every card using forEach, checks which ones have the selected class, reads their dataset.points value using parseInt to convert from text to a number, and adds them to a running total. The total then updates the score number, the progress bar width, the impact level text and colour, the hint message, and the body background image all through direct DOM manipulation.

Impact level thresholds:

  • Low: 0–33 points → Coastline background image
  • Medium: 34–66 points → Mangrove background image
  • High: 67–100 points → Healthy Coral Reef background image

Accessibility

  • All six card images include descriptive alt text relevant to the action being shown
  • Cards use cursor: pointer and a visible hover lift effect so users know they are clickable
  • The selected state adds a visible dark border and fills the radio circle so the selection is clear without relying on colour alone
  • The impact level text label changes between Low, Medium and High so colour is not the only way to communicate the result
  • The progress bar and score number update in real time giving immediate feedback after each selection

Content Page

Technical Description

The Content Page is an educational article about ocean pollution and plastic waste, structured to support meaningful internal navigation across five clearly defined sections.

HTML Structure: The page is divided into a hero banner at the top and a two column layout below using div.content-layout. The left column is a sticky sidebar using Student 1's shared article-sidebar and article-menu classes, containing a numbered ordered list of anchor links. The right column is an <article class="content-main"> containing five content sections each wrapped in a <div> with a unique id. Each section uses <h2> for the section title, <p> for body text, <ul> for bullet points, <div class="callout"> for stat highlight boxes, and <img> with a caption paragraph below.

CSS Techniques: The two column layout uses Student 1's global .content-layout class which applies display: grid with grid-template-columns: 220px minmax(0, 1fr) so the sidebar is a fixed width and the article takes all remaining space. The sidebar uses position: sticky with top: 130px so it follows the user as they scroll down the article. Images use object-fit: cover with a fixed height to keep all images the same size regardless of their original dimensions. The callout box uses a border-left: 4px solid with a light blue background to visually separate key statistics from the body text. The Go to Top button uses position: fixed to keep it visible at all times as the user scrolls.

JavaScript: No JavaScript was required on this page. All interactivity including anchor navigation, the sticky sidebar, and the Go to Top button is achieved purely through HTML anchor links and CSS positioning.

Accessibility

  • All images include descriptive alt text explaining the content of each image
  • Headings follow a logical hierarchy one <h1> for the page title and <h2> for each section
  • Sidebar anchor links allow keyboard users to jump directly to any section without scrolling
  • The Go to Top button is a plain <a> tag making it fully keyboard accessible
  • Sufficient colour contrast is maintained between text and background throughout the page
  • The sticky sidebar keeps navigation visible while scrolling helping users understand where they are in the article

Challenges and Lessons Learned

One of the main challenges I faced was getting the AIS background image to change smoothly when the impact level updated. I initially tried adding a CSS transition on background-image but found that browsers do not animate this property directly. I resolved this by keeping the instant swap and instead adding a smooth transition on the overlay's opacity to soften the visual change.

Another challenge was the Splash Screen content overflowing the screen height on smaller monitors. The h1 was originally set to 90px which pushed the countdown and skip button below the viewport so you had to scroll to see them. I fixed this by reducing the font size and margins step by step until everything fit within 100vh.

On the Content Page I initially used a complex CSS selector .content-main div:first-child h2 to remove the top margin from the first section heading. This was confusing so I simplified it by adding class="first-section" directly to the first div in the HTML which made the intent much clearer.

I also ran into a JavaScript bug on the AIS page where the score was not updating when cards were clicked. After checking the console I found I had written card.datasetpoints instead of card.dataset.points a missing dot that silently broke everything. This taught me to always check the browser console when JavaScript does not behave as expected.

Overall this project taught me that building in phases HTML first, then CSS, then JavaScript makes it much easier to isolate and fix problems because you always know which layer is causing the issue.

Compliance

All pages I implemented comply with JANET regulations governing web page publication on university networks. JANET regulations require that content is appropriate, legal, does not breach copyright, and does not cause harm to others.

  • Appropriate content: All content across my three pages relates directly to SDG 14 and ocean conservation. There is no offensive, harmful, or discriminatory material anywhere on my pages.
  • Copyright compliance: All images used are royalty-free and sourced from appropriate platforms. The Poppins font is loaded from Google Fonts via a standard CDN link which is freely licensed for web use. No copyrighted material has been reproduced without permission.
  • Honest and accurate information: The statistics and facts on the Content Page are based on real environmental data and are presented accurately. I have not exaggerated or fabricated any figures.
  • No malicious code: The JavaScript on my pages is limited to DOM manipulation, timers, and event listeners. No user data is collected, stored, or transmitted to any external server.
  • Accessibility: All pages include alt text on images, semantic HTML structure, and keyboard-accessible elements in line with responsible web publishing standards.
