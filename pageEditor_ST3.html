Feedback & Programme Directory Page

Open live page in a new tab →  |  See validation evidence →

HTML Structure

The page reuses the shared <header>/<nav>/<footer> from the team template. Inside <main>, content is split into semantic <section> elements for the programme directory, the ratings summary, and the reviews. The feedback form uses <fieldset> and <legend> to group "Personal Details" and "Your Feedback" separately.

CSS Techniques

The directory and form sit in a two-column CSS Grid (grid-template-columns: 1.15fr 1fr) that collapses to a single column under 900px using a media query. Programme cards use Flexbox to align the thumbnail image next to the text content. Badges use the global badge colour tokens so they stay consistent if reused elsewhere.

Design and Implementation Rationale

I used a scrollable card list rather than a full grid so the directory doesn't push the feedback form far down the page. The character counter and inline error messages were implemented in JavaScript, per the coursework's requirement for JavaScript-enhanced feedback.

Team Page

Open live page in a new tab →  |  See validation evidence →

HTML Structure

Each team member is a <div class="team-member"> containing an icon, a name, a role, and a hidden "contribution" block. Every card has tabindex="0" and an aria-label describing who the card represents.

CSS Techniques

The hover/reveal effect is done entirely with CSS: the .contribution block starts at max-height: 0 with overflow: hidden, then expands via a max-height transition on :hover, :focus, and :focus-within.

Design and Implementation Rationale

Using :focus-within alongside :hover ensures the reveal works for keyboard users too, not just mouse users — a direct accessibility requirement from the spec.

Content Page

Open live page in a new tab →  |  See validation evidence →

HTML Structure

The article uses one <h1> for the page title and <h2> for each of the four sub-topics, keeping the heading hierarchy logical. Each section has a unique id targeted by the in-page contents list.

CSS Techniques

The "Go to Top" button uses position: fixed so it stays visible while scrolling — implemented with pure CSS, no JavaScript, per the specification.

Design and Implementation Rationale

The in-page contents list was added because the page has four distinct sections and exceeds one screen height, letting readers jump straight to the section they want.

Accessibility Considerations

  • Semantic HTML: used <header>, <nav>, <main>, <section>, and <footer> throughout.
  • Alt text: every image includes descriptive alt text.
  • Keyboard access: team member cards use tabindex="0" and respond to :focus/:focus-within.
  • Form labelling: every field has an associated <label>, grouped with <fieldset>/<legend>.
  • Colour contrast: text colours chosen for readable contrast against the navy/white palette.
